# 扩展阅读

此处记录了官方手册 (opens new window)（中文版 (opens new window)）中包含，但是本书未涉及的概念。

我认为它们是一些不重要或者不属于 TypeScript 的概念，所以这里只给出一个简单的释义，详细内容可以点击链接深入理解。

  • Never (opens new window)（中文版 (opens new window)）：永远不存在值的类型，一般用于错误处理函数
  • Variable Declarations (opens new window)（中文版 (opens new window)）：使用 let 和 const 替代 var，这是 ES6 的知识 (opens new window)
  • this (opens new window)：箭头函数的运用，这是 ES6 的知识 (opens new window)
  • Using Class Types in Generics (opens new window)（中文版 (opens new window)）：创建工厂函数时，需要引用构造函数的类类型
  • Best common type (opens new window)（中文版 (opens new window)）：数组的类型推论
  • Contextual Type (opens new window)（中文版 (opens new window)）：函数输入的类型推论
  • Type Compatibility (opens new window)（中文版 (opens new window)）：允许不严格符合类型，只需要在一定规则下兼容即可
  • Advanced Types (opens new window)（中文版 (opens new window)）：使用 & 将多种类型的共有部分叠加成一种类型
  • Type Guards and Differentiating Types (opens new window)（中文版 (opens new window)）：联合类型在一些情况下被识别为特定的类型
  • Discriminated Unions (opens new window)（中文版 (opens new window)）：使用 | 联合多个接口的时候，通过一个共有的属性形成可辨识联合
  • Polymorphic this types (opens new window)（中文版 (opens new window)）：父类的某个方法返回 this，当子类继承父类后，子类的实例调用此方法，返回的 this 能够被 TypeScript 正确的识别为子类的实例。
  • Symbols (opens new window)（中文版 (opens new window)）：新原生类型，这是 ES6 的知识 (opens new window)
  • Iterators and Generators (opens new window)（中文版 (opens new window)）：迭代器，这是 ES6 的知识 (opens new window)
  • Namespaces (opens new window)（中文版 (opens new window)）：避免全局污染，现在已被 ES6 Module (opens new window) 替代
  • Decorators (opens new window)（中文版 (opens new window)）：修饰器，这是 ES7 的一个提案 (opens new window)
  • Mixins (opens new window)（中文版 (opens new window)）：一种编程模式，与 TypeScript 没有直接关系，可以参考 ES6 中 Mixin 模式的实现 (opens new window)
